    ## Creating the ADT
    <div class="dialogue">
        <div class="stagedir">Betsy and Gamal are planning to build a Contact List in Snap<em>!</em>.</div>
        <div><span>Betsy:</span> We need a way to add a contact to our <var>contact list</var>.</div>
        <div><span>Gamal:</span> Each contact will be a list that with the a person's name, phone number, address, email, or whatever we write.</div>
        <div><span>Betsy:</span> <a href="/cur/programming/2-complexity/2-data-structures-art/2-quizzes.html?topic=nyc_bjc%2F2-conditionals-abstraction.topic&course=bjc4nyc.html&novideo&noassignment" title="In Unit 2, we created a quiz item abstract data type">In Unit 2, we created a <code>quiz item</code> abstract data type</a> to store the questions and answers in a list of quiz items. Here, we can make a <code>contact</code> abstract data type to store the data for each contact in our list of contacts.</div>
        <div><span>Gamal:</span> Yeah. In this project, we'll need a <code>contact</code> constructor and then selectors to access the <code>name</code>, <code>address</code>, and <code>phone number</code> for any given contact.</div>
    </div>

                <div class="takeNote">
                    <div class="sidenote">You learned about input types and output types (domain and range) on <a href="/cur/programming/2-complexity/3-predicates/1-what-is-predicate.html?topic=nyc_bjc%2F2-conditionals-abstraction.topic&course=bjc4nyc.html&novideo&noassignment" title="Unit 2 Lab 3 Page 1: What's a Predicate?">Unit 2 Lab 3 Page 1: What's a Predicate?</a>.</div>
                    <p>It's important to make sure that your inputs to a function match the expected input type. For example, the input type of <code>address from contact</code> matches the output type of <code>contact</code>; they are both of type "contact."</p>
                    <p>If you call <code>address from contact</code> with an input that doesn't match, such as a <em>list</em> of contacts (for example, the <var>contact list</var> variable or the result of running <code>keep</code> and having a subset of that list), it's not going to work. That may sound obvious, but in fact, beginning programmers make mistakes like that all the time; you have to teach yourself to think about the input  and output types of your functions every time you write or use one. </p>
                    <p>That's also true about the inputs to the <code>contact</code> constructor; you can't put something that isn't a name in the name input for <code>contact</code>.</p>
                </div>
